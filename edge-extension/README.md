# 书签同步助手 - Edge浏览器插件

## 简介

书签同步助手是一个Edge浏览器插件，用于将Edge浏览器的书签到同步到TechFunWay书签应用。支持自动同步和手动同步，可配置同步方向和同步间隔，实现真正的双向同步功能。

## 功能特性

- ✅ 自动同步Edge浏览器书签
- ✅ 手动触发同步
- ✅ **支持完整的双向同步**（浏览器 ↔ 后端）
- ✅ **后端删除的书签会同步到浏览器**
- ✅ 可配置同步间隔（1-60分钟）
- ✅ 可配置服务器地址
- ✅ 首次同步支持合并和替换模式
- ✅ 实时显示同步状态
- ✅ 详细的同步日志
- ✅ 性能优化，避免浏览器卡顿
- ✅ 错误处理机制，确保同步过程稳定

## 安装方法

### 方法一：从Edge扩展商店安装（推荐）

1. 打开Edge浏览器
2. 访问Edge扩展商店
3. 搜索"书签同步助手"
4. 点击"获取"按钮安装

### 方法二：从源码安装（开发者模式）

1. 确保已安装Edge浏览器
2. 下载本插件的源码到本地
3. 打开Edge浏览器，输入 `edge://extensions/` 进入扩展管理页面
4. 开启右上角的"开发者模式"
5. 点击"加载已解压的扩展"
6. 选择插件所在的 `edge-extension` 目录
7. 插件会自动安装并显示在扩展栏中

## 配置方法

1. 点击Edge浏览器右上角的插件图标
2. 点击"⚙️ 选项设置"链接
3. 在选项页面中配置以下信息：

### 服务器配置
- **服务器地址**：输入TechFunWay书签应用的后端服务地址，默认为 `http://localhost:8901`

### 同步配置
- **同步间隔**：设置自动同步的时间间隔，范围为1-60分钟，默认为5分钟
- **同步方向**：选择同步方向，可选：
  - 单向同步 (浏览器 → 后端)
  - 双向同步 (浏览器 ↔ 后端)
- **首次同步模式**：选择首次同步时的模式，可选：
  - 合并模式：将浏览器书签合并到后端
  - 替换模式：用浏览器书签替换后端书签

4. 点击"保存配置"按钮保存设置

## 使用方法

### 手动同步
1. 点击Edge浏览器右上角的插件图标
2. 点击"立即同步"按钮
3. 等待同步完成，查看同步状态

### 自动同步
- 插件会根据设置的同步间隔自动执行同步操作
- 同步过程中会在插件图标上显示同步状态
- 同步完成后会更新最后同步时间

### 查看同步状态
1. 点击Edge浏览器右上角的插件图标
2. 在弹出页面中查看：
   - 服务器地址
   - 同步方向
   - 同步间隔
   - 最后同步时间
   - 当前同步状态

## 双向同步说明

### 同步流程
1. **浏览器 → 后端**：将浏览器的书签变更同步到后端
2. **后端 → 浏览器**：将后端的书签变更（包括删除）同步到浏览器
3. **冲突处理**：采用"最后修改优先"策略解决冲突

### 支持的同步操作
- ✅ 创建书签/文件夹
- ✅ 更新书签/文件夹属性
- ✅ **删除书签/文件夹**（双向同步）
- ✅ 移动书签/文件夹
- ✅ 重命名书签/文件夹

## 常见问题

### 1. 同步失败怎么办？

- 检查服务器地址是否正确
- 检查后端服务是否正在运行
- 检查网络连接是否正常
- 查看浏览器开发者工具中的控制台错误信息
- 检查插件的权限是否正确设置

### 2. 插件图标不显示怎么办？

- 确保已将SVG图标转换为PNG格式
- 确保PNG图标文件存在且命名正确（icon16.png, icon32.png, icon48.png, icon128.png）
- 重新加载插件
- 检查插件是否正确安装

### 3. 同步速度很慢怎么办？

- 减少同步间隔
- 减少书签数量
- 确保网络连接稳定
- 插件已实现批量处理机制，每批处理10个操作，避免浏览器卡顿

### 4. 双向同步时出现冲突怎么办？

- 插件默认采用"最后修改优先"策略
- 如有重要书签，建议定期备份
- 查看同步日志了解冲突详情

### 5. 插件一直显示"同步中"怎么办？

- 检查后端服务是否正常运行
- 检查网络连接是否稳定
- 查看浏览器开发者工具中的控制台错误信息
- 尝试重新加载插件
- 如问题持续，可尝试重置插件配置

### 6. 后端删除的书签没有同步到浏览器怎么办？

- 确保同步方向设置为"双向同步"
- 手动触发一次同步操作
- 检查后端服务是否正常运行
- 查看浏览器开发者工具中的控制台日志，了解同步过程

## 技术说明

### 核心功能

- **书签获取**：使用 `chrome.bookmarks.getTree()` 获取浏览器书签
- **书签监听**：使用 `chrome.bookmarks` API 的事件监听器监听书签变更
- **后端通信**：使用 Fetch API 与后端服务通信
- **同步逻辑**：在 `background.js` 中实现完整的同步逻辑
- **配置管理**：使用 `chrome.storage.local` 存储配置信息
- **差异比较**：实现了高效的书签差异比较算法
- **批量处理**：采用批量处理机制，提高同步性能
- **错误处理**：完善的错误处理机制，确保同步过程稳定

### 同步算法

1. **获取书签树**：分别获取浏览器和后端的完整书签树
2. **构建映射**：构建书签和文件夹的映射关系
3. **比较差异**：分析需要创建、更新、删除的项
4. **执行操作**：按批次执行同步操作，确保浏览器响应流畅
5. **更新状态**：更新同步状态和最后同步时间

### 后端API

插件使用以下后端API接口：

- `GET /api/tree` - 获取后端书签树
- `POST /api/import` - 导入书签到后端
- `POST /api/folders` - 创建文件夹
- `POST /api/bookmarks` - 创建书签
- `PUT /api/nodes/{id}` - 更新节点
- `DELETE /api/nodes/{id}` - 删除节点

### 权限说明

插件需要以下权限：

- `bookmarks` - 访问和修改浏览器书签
- `storage` - 存储配置信息
- `alarms` - 设置同步定时器
- `http://localhost:8901/*` - 与后端服务通信
