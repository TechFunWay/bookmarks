# 命令行密码重置工具

## 功能说明

这是一个命令行工具，用于直接在服务器上重置用户密码，适用于管理员忘记密码无法登录的情况。

## 使用方法

### 基本用法

```bash
go run cmd/reset-password.go -username <用户名> -password <新密码>
```

### 参数说明

- `-db`: 数据库文件路径（可选，默认：`./data/db/database.db`）
- `-username`: 要重置密码的用户名（必需）
- `-password`: 新密码（必需，至少6位）

## 使用示例

### 1. 重置管理员密码

```bash
# 重置 admin 用户的密码为 123456
go run cmd/reset-password.go -username admin -password 123456
```

### 2. 指定自定义数据库路径

```bash
# 如果数据库不在默认位置
go run cmd/reset-password.go -db /path/to/database.db -username admin -password 123456
```

### 3. 编译后使用

```bash
# 先编译
go build -o reset-password cmd/reset-password.go

# 使用编译后的程序
./reset-password -username admin -password 123456
```

## 注意事项

1. **安全性**：
   - 此工具只能在服务器本地运行，不能远程访问
   - 建议使用完成后删除或移动到安全位置
   - 使用后建议修改默认密码

2. **密码要求**：
   - 密码长度至少 6 位
   - 建议使用强密码（包含大小写字母、数字和特殊字符）

3. **权限要求**：
   - 需要有访问数据库文件的权限
   - 需要有 Go 运行环境

4. **使用场景**：
   - 管理员忘记密码无法登录
   - 紧急情况下需要重置用户密码
   - 测试和开发环境

## 常见问题

### Q1: 提示"用户不存在"
**A**: 检查用户名是否正确，可以通过数据库查看现有用户：
```bash
sqlite3 data/db/database.db "SELECT id, username FROM users;"
```

### Q2: 提示"连接数据库失败"
**A**: 检查数据库路径是否正确，是否有访问权限：
```bash
ls -la data/db/database.db
```

### Q3: 重置后仍无法登录
**A**: 确认密码长度符合要求，重新尝试。如果问题持续，检查服务器日志。

## 安全建议

1. **使用后清理**：使用完成后，建议删除或移动此工具到非公开目录
2. **定期更换密码**：管理员密码应定期更换，避免长期使用
3. **记录操作**：记录密码重置操作的时间和原因
4. **限制访问**：确保只有授权人员能访问此工具和数据库

## 其他方案

如果需要更安全的密码重置方案，建议实现：

1. **邮箱验证**：通过邮箱发送重置链接
2. **手机验证**：通过短信发送验证码
3. **安全问题**：用户预设安全问题，验证后重置
4. **管理员操作**：在用户管理界面提供重置功能（需要管理员权限）

## 技术细节

此工具的实现逻辑：

1. 双重 MD5 加密（与登录逻辑一致）
2. 直接更新数据库中的密码字段
3. 自动更新 updated_at 时间戳
4. 使用 Go 的标准库和 SQLite 驱动
