# 浏览器书签同步独立窗口功能规格

## Why
目前 Edge 浏览器扩展的书签同步功能集成在 popup.html 弹出窗口中，窗口尺寸受限（300px宽），用户体验不够友好。用户希望将同步功能做成一个独立的页面窗口，提供更宽敞的操作空间和更好的视觉体验。

## What Changes
- 创建新的独立窗口页面 `sync-window.html`，替代原有的 `popup.html`
- 创建对应的 `sync-window.js` 脚本文件
- 修改 `manifest.json`，添加新的页面入口和必要的权限
- 修改 `background.js`，支持从独立窗口发送的消息通信
- 保持原有功能完整：显示同步状态、配置信息、同步按钮、进度条等
- 优化 UI 布局以适应更大的窗口尺寸

## Impact
- Affected specs: Edge 浏览器扩展功能
- Affected code: 
  - `edge-extension/manifest.json`
  - `edge-extension/background.js`
  - `edge-extension/popup.html`（保留作为备选）
  - `edge-extension/popup.js`（保留作为备选）

## ADDED Requirements

### Requirement: 独立窗口页面
The system SHALL provide a standalone window page for bookmark synchronization.

#### Scenario: 打开独立窗口
- **WHEN** 用户点击扩展图标或从选项页面进入
- **THEN** 系统打开一个独立的窗口页面（800x600像素）
- **AND** 页面加载同步配置和状态信息

#### Scenario: 显示同步配置
- **GIVEN** 独立窗口已打开
- **WHEN** 页面初始化完成
- **THEN** 显示服务器地址、同步方式、同步间隔、同步状态等配置信息
- **AND** 显示同步文件夹信息（如果配置了文件夹同步）

#### Scenario: 执行同步操作
- **GIVEN** 独立窗口已打开
- **WHEN** 用户点击"立即同步"按钮
- **THEN** 按钮变为禁用状态并显示"同步中..."
- **AND** 显示进度条和当前同步步骤
- **AND** 同步完成后显示成功或失败状态

#### Scenario: 查看同步历史
- **GIVEN** 独立窗口已打开
- **WHEN** 同步完成后
- **THEN** 显示最后同步时间
- **AND** 显示上次同步的统计结果（创建文件夹数、书签数等）

## MODIFIED Requirements

### Requirement: 扩展清单配置
The manifest.json SHALL support the standalone sync window.

#### Scenario: 页面入口配置
- **WHEN** 扩展加载时
- **THEN** manifest.json 中包含 sync-window.html 的页面定义
- **AND** 保留 action.default_popup 作为备选方案

### Requirement: 后台脚本通信
The background.js SHALL handle messages from the standalone window.

#### Scenario: 跨页面消息通信
- **WHEN** 独立窗口发送同步请求消息
- **THEN** background.js 正确处理并返回响应
- **AND** 同步进度和状态实时更新到独立窗口

## UI/UX 设计

### 窗口规格
- 窗口尺寸：800x600 像素（最小尺寸 600x400）
- 窗口类型：popup（无地址栏、书签栏等浏览器 UI）
- 居中显示

### 样式规范
- 背景色：#f5f5f5
- 主色调：#4CAF50（成功/主要操作）
- 容器背景：白色
- 容器圆角：8px
- 容器阴影：0 2px 10px rgba(0, 0, 0, 0.1)
